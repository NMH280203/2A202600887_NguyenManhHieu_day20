Mục tiêu dự án

Xây dựng hệ thống nghiên cứu đa agent gồm Supervisor, Researcher, Analyst, Writer, Critic, so sánh với single-agent baseline theo các tiêu chí quality, latency, cost. Hệ thống được triển khai trên skeleton Lab 20 với LangGraph, OpenAI, Tavily search, tracing và benchmark tự động.

Deliverables đã hoàn thành:
  • Implement đầy đủ LLM client, Search client, 5 agents, LangGraph workflow
  • CLI: baseline, multi-agent, benchmark
  • Báo cáo benchmark: reports/benchmark_report.md
  • Đánh giá 4 research prompts: reports/prompt_evaluations/
  • Cải tiến Tavily (query truncation) + Critic loop khi citation coverage thấp
  • Trace export tự động: reports/traces/latest.json & latest.md

Kiến trúc hệ thống

User Query → Supervisor → Researcher → Analyst → Writer → Critic → Supervisor → Done

Nếu Critic phát hiện citation_coverage < 0.5, Supervisor gửi lại Writer (tối đa 2 lần revision) trước khi kết thúc.

Cải tiến chất lượng

1. Sửa Tavily API (HTTP 400)

Nguyên nhân: Tavily giới hạn query tối đa 400 ký tự. Prompt dài (hàng nghìn ký tự) gây lỗi 400 → fallback mock example.com.

Giải pháp: Thêm utils/query.py → extract_search_query() — ưu tiên text trong dấu ngoặc kép, câu hỏi, rồi truncate an toàn.

Kết quả: Prompt 2 search thành công với query 85 ký tự → sources thật (arxiv.org, superannotate.com, …).

2. Critic Agent + vòng lặp Supervisor

Luồng mới: … → writer → critic → supervisor. Nếu citation_coverage < MIN_CITATION_COVERAGE (0.5) → Writer revision (tối đa MAX_WRITER_REVISIONS=2).

Test thực tế: routes researcher → analyst → writer → critic → done, coverage = 1.00.

Kết luận

Khi nào nên dùng Multi-Agent?

  • Task nghiên cứu phức tạp cần thu thập → phân tích → viết tách bước
  • Cần trace rõ agent nào làm gì (debug, audit)
  • Cần citation và quality cao hơn, chấp nhận latency/cost
  • Prompt 2 (literature briefing) là use case lý tưởng

Khi nào KHÔNG nên dùng Multi-Agent?

  • Task thiết kế tài liệu (P1, P3, P4) — single-agent dài có thể đạt tương đương
  • Cần response nhanh, cost thấp (chat, FAQ)
  • Task đơn giản không cần decomposition

Hướng phát triển tiếp

  1. Route-aware search: bỏ search cho design tasks, tăng cho research tasks
  2. Bật LANGSMITH_API_KEY để có trace UI trên cloud (local export đã có)
  3. Human evaluation rubric (0-10) thay heuristic tự động
  4. Peer review theo docs/peer_review_rubric.md
